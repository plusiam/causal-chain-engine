# 인과 고리 탐구 엔진

> Causal Chain Inquiry Engine — 데이터로 구동되는 재사용 탐구 도구

학생이 어떤 현상의 인과 구조(**원인 → 과정 → 결과**)를 직접 **구성**하고, 그 고리의 한 지점에 **개입**하면 결과가 달라진다는 것을 추론하게 하는 데이터 기반 탐구 도구입니다. 감염병에서 출발했지만, 내용 데이터만 교체하면 도덕·보건·생태·사회 어디서든 같은 메커니즘으로 재사용됩니다.

**직접 체험** → https://plusiam.github.io/causal-chain-engine/

## 무엇을 하나요

- **① 구성(Build)** — 흩어진 카드를 분류해 인과 고리를 직접 조립합니다. 개념의 구조를 받는 것이 아니라 스스로 세웁니다.
- **② 개입(Intervene)** — 완성된 고리에서 "어느 마디를 건드리면 결과가 바뀔까?"를 추론합니다. 한 지점을 막는 정답은 여럿일 수 있고, 함정 카드는 오개념을 짚어 줍니다.

## 학년·교과

- 대상: 초등학교 3~6학년
- 교육과정: 2022 개정 교육과정 (개념 기반 탐구, 인과관계 렌즈)
- 기본 데이터팩: 보건(감염)·도덕(갈등 조정)·생태(기후 악순환)·사회(미디어 확산)

## 사용 시나리오

1. 교사가 화면 상단 **구성 난이도**를 학년에 맞춰 고릅니다. 3~4학년은 '원인 고정', 5~6학년은 '전부 비움'을 권합니다.
2. 학생이 QR 또는 링크로 접속해 고리를 고르고, 카드를 탭해 자리에 놓아 고리를 완성합니다.
3. 개입 단계에서 고리를 끊을 방법을 **모두** 찾습니다. "막은 방법 N/M" 카운터로 복수 개입 지점을 끝까지 탐색합니다.
4. 교사는 **새 고리 만들기**(에디터)로 직접 주제를 설계하거나, **JSON 내보내기/불러오기**로 동료와 공유합니다.

전자칠판(전체수업)과 모둠 태블릿에서 같은 화면으로 동작하며, 로그인·외부 저장 없이 클라이언트에서만 작동합니다. 학생 데이터는 외부로 전송되지 않습니다.

## 기술

- 단일 HTML 파일, 빌드·의존성 없음 (`index.html`)
- 외부 의존은 웹폰트(CDN)뿐. GitHub Pages 하위 경로에서 바로 동작
- 시나리오는 LocalStorage에 저장, JSON으로 입출력
