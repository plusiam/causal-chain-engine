## 1. 기획 배경 — "수업 콘텐츠"에서 "탐구 도구"로

지금까지 만든 보건 5학년 「감염 고리 만들고 끊기」는 그 자체로 완성도 있는 수업 자료다. 그러나 이 자료를 **한 차시용 콘텐츠**로만 본다면, 들인 설계 노력에 비해 수명이 짧다. 감염병 단원이 끝나면 더 쓰일 곳이 없기 때문이다.

관점을 바꿔 보면 이야기가 달라진다. 우리가 실제로 만든 것은 "감염병 게임"이 아니라, **인과관계라는 개념적 렌즈를 작동시키는 하나의 학습 메커니즘**이다. 이 메커니즘은 감염병과 무관하게 동작하며, 내용 데이터만 교체하면 도덕·보건·생태 어디서든 재사용된다.

이 기획서는 그 도구를 **단발 콘텐츠가 아니라 재사용 가능한 교육 자산**으로 정의하고, 확장·운영·검증의 방향을 제시한다. 이는 147개 이상의 교육용 웹앱을 GitHub Pages로 운영해 온 기존 작업 방식과도 일치한다. 잘 만든 메커니즘 하나가 데이터팩을 갈아끼우며 여러 단원·학년·교과로 번지는 구조다.

## 2. 제품 정의

> **인과 고리 탐구 엔진**은, 학생이 어떤 현상의 인과 구조(원인 → 과정 → 결과)를 직접 **구성**하고, 그 구조의 한 지점에 **개입**하면 결과가 달라진다는 것을 추론하게 하는, 데이터 기반 탐구 도구다.

핵심 메커니즘은 두 동작으로 이루어진다.

**① 구성(Build) — 이해를 만드는 단계.**
학생이 흩어진 요소 카드를 분류해 인과 고리를 직접 조립한다. 개념의 구조를 외부에서 받는 것이 아니라 스스로 세운다.

**② 개입(Intervene) — 적용·전이의 단계.**
완성된 고리에서 "어느 마디를 건드리면 결과가 바뀔까?"를 추론한다. 감염병에서는 이 개입이 "고리 끊기"로 나타나지만, 이는 더 일반적인 **'개입 지점(leverage point) 찾기'의 한 사례**다. 시스템 사고(systems thinking)의 핵심 질문 — "이 구조에서 어디를 바꿔야 가장 효과적인가" — 을 초등 수준으로 번역한 것이다.

이 두 동작은 분리된 게임이 아니라 한 호흡으로 이어진다. **학생이 만든 고리를 학생이 끊는다**는 연결감이 이 도구의 정체성이다.

## 3. 교육적 근거

**개념 기반 탐구(CBC)와의 정렬.**
2022 개정 교육과정이 강조하는 '깊이 있는 학습'은 사실의 암기가 아니라 개념적 이해와 전이를 목표로 한다. 본 엔진의 두 단계는 CBC 탐구 단계와 자연스럽게 대응한다.

| 엔진 동작 | CBC 탐구 단계 | 학생이 하는 사고 |
|---|---|---|
| 구성(Build) | 조사 · 조직(관계 맺기) | 요소들 사이의 인과 관계를 세운다 |
| 개입(Intervene) | 적용 · 일반화 | 구조에 개입해 결과를 바꾸는 원리를 일반화한다 |

**인과관계 렌즈의 작동.**
많은 분류·매칭형 활동은 "A는 B와 짝"이라는 연합 암기에 그친다. 본 엔진은 의도적으로 그 함정을 피한다. 하나의 원인에 복수의 과정·개입을 허용하고, 정답 자리를 미리 표시하지 않으며, 개입 시 "왜 결과가 바뀌는지"를 설명하게 함으로써, 짝짓기가 아니라 인과 *추론*이 일어나게 한다.

**전이(transfer)를 겨냥한 설계.**
같은 메커니즘이 여러 주제로 반복되면, 학생은 표면적 소재(감염병·갈등·기후)가 달라도 "구조를 세우고 개입 지점을 찾는다"는 사고 방식을 학습하게 된다. 이는 IB PYP가 강조하는 개념적 이해의 전이와도 맞닿는다.

## 4. 콘텐츠 데이터 모델

엔진의 재사용성은 **내용과 동작의 분리**에서 나온다. 동작(조립·개입·채점·피드백)은 고정된 코드가 담당하고, 내용은 시나리오 데이터가 담당한다. 교사는 코드를 건드리지 않고 데이터만 교체해 새 주제를 만든다.

현재 v1의 `SCENARIOS` 구조를 도메인 중립적으로 일반화하면 다음과 같다.

```
시나리오(고리) = {
  주제명, 부제,

  // ① 구성 단계: 학생이 분류해 조립할 카드
  원인:   { 아이콘, 이름 },        // 고리의 출발(고정 앵커)
  과정:   { 아이콘, 이름 },        // 전파·매개 등 중간 마디
  결과:   { 아이콘, 이름 },        // 귀결(병·관계 단절·환경 악화 등)

  // ② 개입 단계: 완성된 고리에서 끊을(개입할) 지점
  마디:   [ 원인, 과정, 결과 ],     // 고리를 이루는 3개 마디
  개입지점: [
    { 통하는행동:[행동id…], 설명:"왜 이 지점을 건드리면 결과가 바뀌는가" },
    …
  ],

  // 행동 카드 (정답 + 함정을 섞음)
  행동:   [ { id, 아이콘, 이름 } … ],

  // 함정 카드의 오개념 교정 문구
  힌트:   { 행동id : "왜 이 행동으론 안 되는지" }
}
```

이 구조의 두 가지 설계 원칙이 곧 교육 효과의 핵심이다.

- **한 지점에 정답이 여럿** — 하나의 개입지점을 여러 행동으로 막을 수 있게 한다. "정답은 하나가 아니다"를 반복 경험하게 만든다.
- **함정 카드는 오개념 교정 장치** — 무관하거나 해로운 행동을 일부러 섞고, 선택 시 그 이유를 짚는다. (예: 눈병에 마스크, 모기 감염에 손씻기 → "그 방법이 늘 정답은 아니다"를 가르친다.)

## 5. 적용 사례 맵

같은 엔진에 다른 데이터를 넣었을 때의 확장 예시다. 모든 사례가 "원인 → 과정 → 결과"의 고리와 "개입 지점"을 가진다.

| 교과 | 고리(원인 → 과정 → 결과) | 개입 지점 예시 | 함정(오개념 교정) |
|---|---|---|---|
| **도덕** 정직 | 작은 거짓말 → 들킬 위기 → 더 큰 거짓말 → 신뢰 붕괴 | 초기에 솔직히 말하기 · 중간에 사과하기 | "더 그럴듯한 거짓말로 덮기"(악화) |
| **도덕** 갈등 조정 | 사소한 오해 → 감정적 말 → 맞받아치기 → 관계 단절 | 잠시 멈추기(쿨다운) · 입장 바꿔 듣기 · 먼저 사과 | "똑같이 되갚기"(악화) |
| **도덕** 공동체 | 방관 → 동조 → 괴롭힘 강화 → 피해 심화 | 방관자가 목소리 내기 · 도움 요청 · 신뢰하는 어른에게 알리기 | "못 본 척하기"(방관) |
| **보건** 안전 | 부주의 → 위험 행동 → 사고 → 부상 | 안전 규칙 지키기 · 보호장구 착용 | "괜찮겠지 하고 넘기기"(악화) |
| **생태** 기후 | 온실가스 배출 → 기온 상승 → 빙하 융해 → 추가 가열(악순환) | 배출 줄이기 · 에너지 전환 · 숲 보전 | "나 하나쯤은 괜찮다"(악화) |
| **사회** 미디어 | 자극적 정보 → 무비판 공유 → 빠른 확산 → 오해 확산 | 출처 확인하기 · 공유 멈추기 | "재밌으니까 일단 공유"(악화) |

도덕 영역은 가장 강력한 적용처이자 가장 신중해야 할 영역이다. 괴롭힘·갈등 같은 주제는 반드시 **가상 시나리오로 구성**하고, 실제 사건·실명을 다루지 않으며, '가해자 처벌'이 아니라 '고리를 끊는 방관자의 힘'에 초점을 둔다. (→ 9장 위험 관리 참조)

## 6. 시스템 구조

- **단일 HTML 파일** — HTML·CSS·JS를 한 파일에 담아 의존성과 빌드 과정을 없앤다. GitHub Pages(plusiam)에 올리면 즉시 배포된다.
- **내용/동작 분리** — v1은 `SCENARIOS`가 코드 상단에 있으나, v2에서 외부 JSON으로 분리해 교사가 코드를 보지 않고도 주제를 추가하게 한다.
- **반응형(fluid)** — `clamp()` 기반 자동 크기 조절로 전자칠판(대형)과 모둠 태블릿(중형)을 한 코드로 대응한다.
- **조작 방식** — 드래그가 아닌 '카드 탭 → 자리 탭'. 대형 터치스크린·마우스·태블릿에서 동일하게 안정적이다.
- **오프라인·무계정** — 외부 저장소나 로그인 없이 동작해 교실 네트워크 제약에서 자유롭다. (학습 기록 연동은 v3의 선택 기능)
- **라이선스** — CC BY-NC 4.0, 저작권자 표기 내장.

## 7. 개발 로드맵

**v1 — 검증된 프로토타입 (완료)**
감염 고리 1주제(4시나리오), 구성+개입 두 단계, 반응형, 탭 조작, 함정 카드·피드백·채점. 데이터는 코드 내 하드코딩.

**v2 — 엔진화 + 교사 도구**
- 시나리오 데이터를 외부 JSON으로 분리, 스키마 문서화
- **교사용 시나리오 에디터** — 폼에 원인·과정·결과·개입지점을 입력하면 고리가 생성되는 도구
- 주제 갤러리(여러 교과 데이터팩 선택)
- 접근성 보강 — 키보드 조작, ARIA 레이블, 저시력 대비(명도·크기)

**v3 — 학생 창작 + 수업 연계**
- **학생 창작 모드** — 학생이 직접 고리를 설계·공유(메이커 활동, 평가 자료로 활용)
- 만든 고리를 이미지로 내보내 수업 기록(Obsidian PKM)과 연동
- 모둠 점수판·협동 모드(전자칠판 전체수업용)
- 다국어 — 다문화·이음교육 맥락 대응

## 8. 적용 및 검증 계획

이 도구는 **형성평가 장면**으로 그대로 쓰인다.

- **구성 단계 = 이해 확인** — 고리를 올바르게 조립하는가로 개념 구조의 이해를 본다.
- **개입 단계 = 인과 추론 평가** — "왜 그 지점을 끊었는가"의 발화가 곧 평가 증거가 된다. 이는 보건 성취기준 [6보04-03]의 '전파 경로와 예방 수칙을 연결지어 설명한다'와 동작이 일치한다.

**관찰 포인트(루브릭 축).**
① 단순 매칭에 그치는가, 인과로 설명하는가 ② 개입 지점을 하나만 찾는가, 복수로 찾는가 ③ 함정 카드 앞에서 오개념을 스스로 교정하는가.

**전이 확인(간단 사전·사후).**
수업에서 다루지 않은 새 고리를 제시하고 개입 지점을 찾게 한다. 소재가 달라도 사고 방식이 옮겨 가는지(전이) 확인한다.

## 9. 위험 및 제약

- **분량(차시) 과다** — '구성 4 + 개입 4 = 8라운드'는 한 차시에 빠듯하다. 시나리오 수를 2~3개로 줄이거나, 구성은 전자칠판 전체수업·개입은 모둠 태블릿으로 역할을 나눈다. *(이는 우리가 수업 지도안 피드백에서 강조한 '선택과 집중'을 도구 설계에 그대로 적용한 것이다.)*
- **정서적 민감 주제** — 도덕의 괴롭힘·갈등은 가상 시나리오로만 다루고, 실제 사건·실명 배제, 정서적 안전을 최우선으로 한다.
- **보건의 민감 영역** — 식이·체중 등은 구체적 수치·목표치를 다루지 않으며, 일반적 '건강한 생활' 수준에서만 구성한다.
- **단선적 인과의 오개념** — 실제 인과는 다요인·순환적이다. '한 원인-한 결과'로 단순화될 위험을, 복수 원인·복수 개입 장치와 (고학년) 순환 고리 도입으로 완화한다.
- **접근성** — 현재 탭 조작으로 드래그 장벽은 낮으나, 키보드·스크린리더 지원은 v2에서 보강한다.
